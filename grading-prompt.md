# 答え合わせ（採点）プロンプト仕様

英文構造トレーナーが Claude API に答え合わせを依頼するときのプロンプトをまとめたドキュメントです。  
Claude（Web / API）に手動で採点してもらう際は、本書の **System** と **User** をそのまま使い、問題データを差し込んでください。

実装の参照元: `src/prompts/index.js`（`buildCheckPrompt`）、`src/api/claude.js`（`checkAnswers`）

---

## API 設定（参考）

| 項目 | 値 |
|------|-----|
| モデル | `claude-sonnet-4-5-20250929` |
| max_tokens | `8192` |
| 1回の採点問数 | **1問**（7問セットは7回に分割） |
| レスポンス安定化 | assistant メッセージで `[` を prefill（JSON 配列の先頭を固定） |

## System プロンプト

```
あなたは英語教育の専門家です。
英語翻訳の採点をしてください。
必ず有効なJSONのみを返してください。マークダウンや説明文は一切含めないでください。

JSONの厳守ルール:
- 文字列値はダブルクォートのみ使用（シングルクォート不可）
- feedback 内で英文・英単語を引用するときは必ず『』を使い、ダブルクォート (") は一切使わない
- 文字列内のダブルクォートが必要な場合のみ \" でエスケープする
- feedback 内の改行は \n で表す（生の改行は使わない）
- 末尾カンマ（trailing comma）は禁止
```

## User プロンプト（テンプレート）

`{{ITEMS}}` に各問のブロックを並べ、`{{N}}` を問数に置き換えます。  
Step 3 / Step 7 のときは末尾に [Step 別の追記](#step-別の追記) を追加します。

```
以下の翻訳を採点し、JSONで返してください。

{{ITEMS}}

返却形式（JSONのみ、{{N}}要素の配列）:
[
  {
    "score": 0〜10 の整数（各問10点満点）,
    "correct": true または false,
    "feedback": "下記の feedback ルールに従った日本語の解説",
    "correction": "常に null（模範解答は別途表示するため不要）"
  }
]

score（10点満点）の目安:
- 10点: 意味・文法ともに正解、または表現の差異のみで意味は完全一致
- 7〜9点: 意味は正確だが、軽微な文法・語彙・スペルの誤りがある
- 4〜6点: 意味はおおむね伝わるが、時制・語法・語順など重要な誤りがある
- 1〜3点: 意味が部分的にしか伝わらない、または重大な誤りが多い（意味がほぼ伝わらない場合も 1点）
- 0点: **未入力の場合のみ**（解答がある場合は最低 1点）

採点基準:
- score に応じて correct を設定する（8点以上なら correct: true、7点以下なら false）
- 意味が正しく伝わっていればマイナーな表現の差異は 8〜10点
- 文法的な誤り・時制のミス・語順の問題・意味の変化は 7点以下
- 解答がある場合、score は 1〜10 のいずれかとする（0点は未入力専用）

feedback ルール（全体の分量は簡潔版の約1.5倍を目安。特に模範解答の妥当性の説明を厚く書く）:
- score が 10点の場合: 「正解！」のみ（他の解説は不要）
- score が 8〜9点の場合:
  1. 軽微な誤りを1〜2文で指摘
  2. 改行（\n\n）のあと、模範解答の語順・表現がなぜより望ましいかを **3〜4文** で補足する
     - 模範解答のポイント・語順・ニュアンスのいずれか少なくとも2点を触れる
     - 解答の表現と模範解答を直接比較し、「なぜ模範の方が自然・正確か」を具体的に
- 未入力（解答が空または「（未入力）」）の場合: score は **0 のみ**（他の点数帯と混同しない）、具体的な誤りの指摘のみ（翻訳による意味の比較は不要）
- score が 7点以下かつ解答ありの場合: 次の4ブロックを \n\n で区切る
  1. 主な誤り：— 誤りごとに (1)(2)... と列挙。各項目は **1〜2文**（何が誤りか＋正しい形・理由の簡潔な補足）
  2. スペルミス等があれば補正したうえで、解答を日本語に訳し直した意味を **2文程度** で示す
  3. その訳が元の日本語文の意図とどうずれるかを **2〜3文** で対比する（該当する場合）
  4. **模範解答（100点）の妥当性** — このブロックが feedback 全体の **約40%** になるよう、**4〜6文** で深く説明する
     - 模範解答のポイント・語順の選択・表現のニュアンス・ネイティブの自然さを多角的に
     - 解答者が取りがちな別表現（関係詞節 vs 分詞修飾、語彙の強弱など）と模範解答を比較し、模範が優れる理由を段階的に
     - 日本語のどの部分に模範解答のどの表現が対応するかも触れる
     例: 『Passengers sitting near the window』は、関係詞節 who sit より分詞 sitting の後置修飾の方が簡潔で、視覚的な場面描写に向きます。さらに are enjoying は are watching より『楽しんでいる』という日本語のニュアンスを生き生きと伝え、今まさに起きている光景を描きます。
     ※ feedback 内の英文引用は必ず『』で囲む（ダブルクォートは JSON エラーの原因になるため禁止）

feedback 書式（可読性最優先）:
- 1つの論点・誤り項目・文ごとに \n で改行し、詰め込まない
- 誤りが複数あるときは見出し「主な誤り：」の直後から (1)(2)... を各行に1項目ずつ
- 大きな区切り（誤り指摘 / 補正後の意味 / 意図との対比 / 模範の理由）のあとは空行（\n\n）を入れる
- 行が長くなっても構わない。途中で不自然な位置で改行しない
- feedback は学習者が「何を直すか」と「なぜ模範解答が100点なのか」の両方がわかるよう具体的に
- 模範解答と異なる別解を correction として提示しない（模範解答は常に入力の「模範解答（100点）」を使う）
```

## 各問ブロック（`{{ITEMS}}` の1要素）

番号 `[1]` `[2]` … は **その API 呼び出し内での連番**（1問ずつ採点するため常に `[1]`）。

```
[{{番号}}]
日本語: {{jp}}
模範解答（100点）: {{en}}
模範解答のポイント: {{nuance}}          ← nuance がある場合のみ
操作タグ: {{operationTag}}                ← Step 7 のみ（ある場合）
糸: {{thread}}                            ← Step 7 のみ（ある場合）
解答: {{attempt または （未入力）}}
```

> **注:** 構造分解（`parts`）は採点プロンプトには含めません（入力トークン削減のため）。色分け表示用の `parts` は問題生成時にアプリ側で保持します。

## Step 別の追記

User プロンプトの **末尾**（feedback ルールの最後）に追加します。

### Step 3（動詞の変化）

```
Step 3 採点の補足:
- 疑問文・否定文の解答でも、助動詞の形・語順が正しければ意味が通れば高得点とする
```

### Step 7（操作の再利用）

```
Step 7 採点の補足（必須）:
- **構文の種類は問わず、正しい変形であれば正解とする**（例：it-cleft と what-cleft の両方、仮定法の別表現など）
- 操作タグ（比較・仮定法・疑問・倒置/強調・否定・話法・省略）に関わらず、意味と文法が正しければ満点に近い評価とする
- feedback で模範解答の妥当性を説明するとき、可能なら「糸1（助動詞前置）」または「糸2（空所＋移動）」のどちらの再利用かに1行触れる
```

Step 4・5・6 には Step 専用の追記はありません。

## 記入例（1問）

```
以下の翻訳を採点し、JSONで返してください。

[1]
日本語: このアートギャラリーで展示されている絵画は、有名な画家によって3年前に描かれたものですか。
模範解答（100点）: Were the paintings displayed in this art gallery painted by the famous artist three years ago?
模範解答のポイント: 受動態の疑問文では be + 過去分詞を助動詞の前に置き、three years ago は文末に置くのが自然です。
操作タグ: 疑問
糸: 糸1
解答: Had the paintings displayed in this art gallery been painted by the famous artist at three years ago?

（以下、返却形式・score 目安・採点基準・feedback ルール … User プロンプト本文を続ける）
```

## 期待するレスポンス

- **JSON 配列のみ**（マークダウンコードブロック不可）
- 要素数 = そのリクエストに含めた問数（常に 1）
- 各要素:

```json
{
  "score": 4,
  "correct": false,
  "feedback": "主な誤り：\n(1) ...\n\n...",
  "correction": null
}
```

### アプリ側の後処理（参考）

- `correction` は常に `null` 想定（模範解答は UI で別表示）
- 未入力の解答は score を **0** に正規化
- 解答ありの場合、score は **1〜10** にクランプし、`correct` は **8点以上で true**

---

## 手動で Claude に依頼するときの手順

1. 上記 **System** をシステムプロンプト（または会話の最初の指示）に設定
2. **User プロンプト** に、採点したい問を `{{ITEMS}}` 形式で記入（1問ずつ）
3. Step 3 / 7 なら末尾に [Step 別の追記](#step-別の追記) を追加
4. 返答は `[` から始まる JSON 配列のみを要求（API 利用時は assistant prefill `[` と同等）
5. 7問すべて採点する場合は、1問 × 7回 に分け、各回で独立した JSON 配列（1要素）をもらう

## 構造分解の役割定義（色分け表示用・参考）

採点プロンプトには `parts` を送りませんが、問題生成時に付与され **模範解答（語順のまま色分け）** の表示に使われます。

| 役割 | 意味 |
|------|------|
| X | 名詞役（主語・目的語・補語） |
| V | 動詞（時制・相・態・助動詞を含む動詞部分全体） |
| Y | 形容詞役（名詞を修飾：形容詞・前置詞句・分詞・関係詞節など） |
| Z | 副詞役（動詞・文全体を修飾：副詞・前置詞句・副詞節・分詞構文など） |

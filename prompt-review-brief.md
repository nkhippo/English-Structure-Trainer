## 相談依頼文（そのまま使えます）

> 以下は「英文構造トレーナー」という英語学習 Web アプリの背景・現行仕様・問題生成プロンプトです。  
> **目標:** 7問セットの初回作成時に、**疑問文の出題を増やしたい**（理想は 7問中 2問程度が疑問文）。ただし **jp の自然さ・Step の文法焦点・MECE 網羅・parts/inner の品質を下げたくない**。  
> UI でユーザーが疑問文数を調整する方式は一旦見送りました。**プロンプト設計の改善案**を中心にレビューしてください。  
> 具体的に知りたいこと:  
> 1. Step 3〜7 それぞれで、疑問文を増やすためのプロンプト文言・制約の付け方  
> 2. 「少なくとも2問は疑問/否定」と「ちょうどN問は疑問文」のどちらが学習効果・品質のバランスに良いか  
> 3. 否定文と疑問文を分けてカウントすべきか  
> 4. Step 4〜6（関係詞・ネスト等）で疑問文を増やすときの設計上の注意（平叙文の文法焦点を損なわない方法）  
> 5. Step 7 の operationTag「疑問」と Step 3 の疑問文練習の役割分担  
> 6. テーマ割り当て・MECE 網羅・トークン上限とのトレードオフ  
> 7. 弱点克服（5/3/1問）では疑問文数を指定しない方針でよいか

## 1. このアプリの目的（要約）

開発者は英語留学中。論理的思考は得意だが、英語の「文法・構造のお作法」を体系的に積み上げてこなかった。

**自覚している3課題:**

| # | 課題 | 症状 |
|---|------|------|
| ① | 骨格が作れない | S+V(+O/C) を即座に決められず詰まる |
| ② | 句・節のフレームが曖昧 | 不定詞・動名詞・関係詞などの配置が固まっていない |
| ③ | ネストで崩れる | 関係詞節（Y）+ 副詞節（Z）の同時処理で崩れる |

**学習フロー:**

```
日本語を読む → 自分で英語を組み立てる → まとめて答え合わせ → 構造分解で理解
```

- Step 3〜7 の翻訳練習（1セット **7問**）+ フレーズ練習タブ（別系統）
- 採点は **en（文法・構造の模範）のみ** が100点基準。enNative は参考表示
- 弱点克服: 7→5→3→1 と段階的に再出題（Markdown アップロードからも再開可）

## 2. X / Y / Z フレーム（構造分解の前提）

| 記号 | 役割 | 典型 |
|------|------|------|
| **X** | 名詞役（S・O・C） | 名詞 / 動名詞 / 不定詞 / that節 / wh節 |
| **V** | 動詞（時制・相・態・助動詞全体） | is reading / has been written |
| **Y** | 形容詞役（名詞の後置修飾） | 形容詞 / 前置詞句 / 分詞 / 関係詞節 |
| **Z** | 副詞役 | 副詞 / 前置詞句 / 副詞節 / 分詞構文 |

`parts` / `inner` で再帰的に分解し、採点後に UI で色分け表示する。

## 3. STEP 一覧と疑問文の位置づけ

| Step | 焦点 | 疑問文との関係（教科書・エッセンス） |
|------|------|--------------------------------------|
| **3** | 時制・相・態・助動詞 + **疑問・否定の基本** | 糸1=助動詞前置 / 糸2=空所を文頭へ。**疑問と否定は対称** |
| **4** | 準動詞・前置詞句（役割≠形） | 疑問文の明示制約なし |
| **5** | 関係詞・後置修飾 | 疑問文の明示制約なし（wh節は名詞節として別扱い） |
| **6** | 節と接続・ネスト | 疑問文の明示制約なし |
| **7** | 比較・仮定法・**疑問**・倒置/強調・否定・話法・省略 | operationTag「疑問」で疑問文体系全体（間接疑問・付加疑問等） |

## 4. 現行の問題生成仕様

### 4.1 API

| 項目 | 値 |
|------|-----|
| モデル | `claude-haiku-4-5-20251001` |
| 新規 | 7問 / 弱点克服は前回−2問 |
| テーマ | 18テーマから各問1つ（重複回避） |
| シード | `steps.js` からランダム2問を参考例として注入 |

### 4.2 全 Step 共通

- User プロンプトに **STEP_ESSENCE**（再利用原理）と **STEP_COVERAGE**（MECE 網羅）を注入
- `nuance` 末尾にエッセンス参照を1行必須
- `enNative` は生成しない（UI オンデマンド）
- 初回作成と弱点克服でプロンプトが分岐（後者は Markdown + core タグ集計を末尾に付与）

### 4.3 疑問文に関する現行プロンプト指示

#### Step 3（`buildStep3GenerateExtra`）

```
- {n}問のうち **少なくとも2問** は疑問文または否定文を含める
- 疑問文：Yes/No疑問と wh疑問の **両方** をセット内でカバー
- 否定文：助動詞 + not
- jp に疑問・否定の手がかりを自然に含める
```

加えて `STEP_COVERAGE[3]`:

```
- うち1問は wh疑問（糸2の予告編）
- 時制×相をサンプリング網羅
```

#### Step 4〜6

- 疑問文数に関する指示 **なし**
- 各 Step の MECE 網羅（役割対比・関係詞カテゴリ・ネスト等）のみ

#### Step 7（`buildStep7GenerateExtra`）

```
- 各問に operationTag を1つ（比較/仮定法/疑問/倒置・強調/否定/話法/省略）
- 1セットで 2〜3種類の operationTag を混在
- 最低1問は「倒置/強調」+ 否定副詞句による倒置
```

「疑問」タグは7種のうちの1つ。2〜3種混在のため、疑問文は **多くても1〜2問程度** が期待値。

### 4.4 弱点克服時

- 疑問文数の指定 **なし**
- core 誤りタグ（skeleton / verbInfo / role / attachment / nesting / advanced）に基づき弱点を厚く出題
- 出題 Step は現在の Step に限定

## 5. 現状の課題（なぜ疑問文が少ないか）

| 要因 | 説明 |
|------|------|
| **Step 3: 疑問と否定の合算** | 「少なくとも2問は疑問**または**否定」→ 否定文2問で条件を満たし、疑問文は0〜1問になりうる |
| **Step 4〜6: 指示なし** | 平叙文がデフォルト。Step 焦点（関係詞・ネスト等）が優先される |
| **Step 7: タグの希薄化** | operationTag が7種 + 倒置必須1問 → 「疑問」タグが当たる確率は低い |
| **品質制約との競合** | テーマ多様化・MECE 網羅・jp 自然さ・文長上限が同時に要求され、疑問文は後回しになりやすい |
| **モデル** | Haiku で7問一括生成。複合制約の優先順位が曖昧だと平叙文に寄る |

**観測:** 実運用では7問中、疑問文が **0〜1問** 程度にとどまることが多く、Step 3 の学習目的（助動詞操作＝疑問・否定）の練習量が不足しがち。

## 6. 検討したが見送った案

### UI スライダー（リバート済み）

- 初回作成時のみ、疑問文数を 1〜7 で指定（デフォルト 2/7）
- プロンプトに「ちょうど N 問は疑問文」を注入
- **見送り理由:** プロンプト品質・Step 4〜7 との整合を先に詰めたい。UI は後回し

## 7. 採点側の制約（疑問文を増やすときの前提）

`buildCheckPrompt` の Step 3 追記:

```
疑問文・否定文の解答でも、助動詞の形・語順が正しければ意味が通れば高得点とする
```

Step 7 追記:

```
構文の種類は問わず正しい変形であれば正解。operationTag に関わらず意味と文法が正しければ満点に近い評価
```

→ 生成側で疑問文を増やしても、採点は **助動詞の形・語順** を重視する設計。模範 `en` 基準は維持。

## 8. MECE 網羅との関係（essences 要約）

| Step | 網羅規則（疑問文以外も同時に満たす必要あり） |
|------|---------------------------------------------|
| 3 | wh疑問1問 + 時制×相サンプリング + 疑問/否定最低2問 |
| 4 | 同形異役割2問 + minimal pair 1組 |
| 5 | 関係代名詞・関係副詞・what節・同格that を別カテゴリ |
| 6 | Y+Z共起キーセンテンス1問 + 副詞節・名詞節・等位 |
| 7 | operationTag 2〜3種混在 + 倒置必須1問 |

疑問文を増やす提案は、**上記 MECE を壊さない** ことが条件。

## 9. 相談時に特に見てほしい観点

### 9.1 プロンプト設計

1. **Step 3:** 「疑問文ちょうど2問 + 否定文は別カウント」とすべきか。wh / Yes-No の配分は固定すべきか
2. **Step 4〜6:** 疑問文を入れる価値はあるか。入れるなら「その Step の文法焦点を疑問文の中で練習する」具体例（例: Step 5 で関係詞を含む wh疑問）
3. **Step 7:** operationTag「疑問」の最低保証（例: 最低2問）と、他タグ（倒置必須等）の両立
4. **生成手順の明示:** 「先に疑問文スロットを2問確保してから残りを設計」など、手順レベルの指示は有効か
5. **二段生成:** 7問一括 vs 疑問文2問+平叙5問の分割生成 — 品質・コストのトレードオフ

### 9.2 品質ガードレール

6. jp に「〜ですか」を入れる指示と、自然な和文のバランス
7. 疑問文が増えると vocabHints / parts / inner の JSON が肥大化しやすい — 緩和策
8. シード例2問が平叙文ばかりだと疑問文が減る — 参考例の選び方

### 9.3 学習設計

9. Step 3 で疑問文を厚くし、Step 7 の「疑問」タグは発展（間接疑問・付加疑問）に専念する分担は妥当か
10. 弱点克服では疑問文数を指定しない現方針でよいか

# 英文構造トレーナー — サービス仕様（STEP別）

## 0. アプリ全体の目的

### 0.1 解決したい学習課題

| # | 課題 | 内容 |
|---|------|------|
| ① | 骨格が作れない | S+V(+O/C) を即座に決められず、英作文・スピーキングで詰まる |
| ② | 句・節のフレームが曖昧 | 不定詞・動名詞・分詞・関係詞・副詞節など「どの形で・どこに置けるか」が固まっていない |
| ③ | ネストで崩れる | 関係詞節（Y）と副詞節（Z）が骨格にぶら下がると一気に処理できなくなる |

### 0.2 学習アプローチ

- **日本語 → 英語の翻訳** という能動的アウトプットで構造を鍛える
- 採点後に **X / Y / Z / V の構造分解** と照合し、「なぜその語順・表現か」を理解する
- 関連アプリ群の **第3レイヤー**（単語 → 例文 → **構造**）

### 0.3 採用フレームワーク（X / Y / Z / V）

```
① 骨格（文型）= S + V (+O / +C)
     └ 動詞は4情報：時制・相・態・助動詞
② 骨格の各位置を「かたまり（句・節）」が埋める／飾る
     かたまりは必ず X / Y / Z のいずれか
③ 骨格が複数あるとき → 接続でつなぐ
```

| 記号 | 役割 | 典型の形 |
|------|------|----------|
| **X** | 名詞役（S・O・C） | 名詞 / 動名詞 / 不定詞 / that節 / wh節 |
| **V** | 動詞（時制・相・態・助動詞を含む動詞部分全体） | is reading / has been written / can speak |
| **Y** | 形容詞役（名詞を後置修飾） | 形容詞 / 前置詞句 / 分詞 / 関係詞節 / 同格節 |
| **Z** | 副詞役（動詞・文を修飾） | 副詞 / 前置詞句 / 不定詞（目的）/ 分詞構文 / 副詞節 |

**再帰性**：句・節の内部にも同じ X/Y/Z ルールが適用される（`inner` で表現）。

### 0.4 ステップ設計の原則

```
骨格 → 動詞情報 → 句がぶら下がる → 節がぶら下がる → 節どうしがつながる
```

| Step | 状態 | 内容 |
|------|------|------|
| 1 | 前提（学習済み） | 品詞 |
| 2 | 前提（学習済み） | 文型・自他動詞 |
| **3** | **アプリ対象** | 動詞の変化 |
| **4** | **アプリ対象** | 準動詞・前置詞句 |
| **5** | **アプリ対象** | 関係詞 |
| **6** | **アプリ対象** | 節と接続・ネスト |
| **7** | **アプリ対象** | 比較・仮定法・倒置など（発展構文） |

## 1. 全STEP共通のサービス仕様

### 1.1 セッションの流れ

1. **APIキー入力**（初回のみ）→ `localStorage` に保存
2. **Step タブ選択**（3 / 4 / 5 / 6 / 7）
3. **「問題を作成する」** → Claude API で **7問** 生成
4. 各問で **日本語文を読む** → **英訳を入力**（全問入力してから採点）
5. **「まとめて答え合わせ（N問）」** → Claude API が1問ずつ採点（`errorTags` 付与）
6. 採点後に各問で **模範解答・構造分解・フィードバック** を表示
7. **弱点克服の再出題**（7→5→3→1）または **Markdown 読み込み** から続き
8. **enNative** はボタン展開時にオンデマンド生成（生成バッチには含めない）

### 1.2 UX上の設計判断

| 方針 | 理由 |
|------|------|
| 全問まとめて出題 → まとめて採点 | 1問ずつだと答えを見て能動性が下がる。集中を維持しやすい |
| 構文ヒントは出さない | 思考の起点を「自分で骨格を組む」に固定（active recall） |
| 単語ヒントのみ任意表示 | 語彙で止まらず、構造練習に集中させる |
| 採点後に構造を一括振り返り | 色分け・骨格フレーム・各チャンクの役割を同時に確認 |

> **注**：README / `background.md` に「できた / 要復習」の自己採点が記載されているが、**現行コードには未実装**。

### 1.3 1問あたりのデータ構造（Exercise）

```json
{
  "jp": "自然な日本語文",
  "en": "100点満点の模範英訳",
  "parts": [
    {
      "t": "英文チャンク",
      "r": "X|V|Y|Z",
      "n": "役割メモ · 語順・表現の理由",
      "inner": [ /* 再帰的に同構造、最大2段 */ ]
    }
  ],
  "nuance": "模範解答が100点となる理由",
  "vocabHints": [{ "jp": "辞書形", "en": "原形・単数形" }]
}
```

**parts の整合性ルール**

- トップレベルの `parts[].t` をスペース連結 → `en` と完全一致
- `inner[].t` をスペース連結 → 親 `t` と完全一致
- 句・節を含むチャンクには `inner` 必須
- 単純な主語・動詞・単語副詞などは `inner` 省略可

### 1.4 問題生成プロンプトの共通要件

#### 生成手順（順番固定）

1. まず **自然な日本語 `jp`** を書く
2. `jp` を正確に英訳して **`en` と `parts`** を作る
3. 英文の構文要件のために **jp を英語語順に無理やり合わせない**

#### 日本語（jp）の品質要件

- 和文として自然（カクル調・直訳調を避ける）
- 意味の論理的一貫性（例：×「去年出版された著者」）
- 関係詞・後置修飾の Step では **連体修飾**（名詞の前に修飾句）を使う
- 複数節をつなぐとき論理関係を明示（並列・因果など）
- 後節の述語は文脈なしで意味が通るように書く

#### 模範解答（en）の品質要件

- 「意味が通る訳」ではなく **100点満点の最良訳**
- 訳し方が複数ある場合、**当該 Step の文法ポイントを最も活かす語順・表現** を選ぶ

#### vocabHints のルール

- TOEIC 350点程度で思い浮かびにくい語を 2〜4 語
- 基本語（book, go, tired 等）は含めない
- 接続詞・関係詞は含めない（文法ヒント禁止）
- 動詞原形・名詞単数形で記載
- 該当なしは `[]`

#### テーマの多様性

- 18テーマプールから各問に異なるテーマを割り当て
- 同セット内でテーマ・場面・主語・文型の重複を避ける
- 生成後に並びをランダム化

#### 参考例（シード）の使い方

- `steps.js` の静的シードから **2問をランダム抽出** し、文体・自然さの基準として提示
- テーマ・内容は参考例に引きずられない

### 1.5 採点プロンプトの共通要件

- **10点満点** × 7問 = 最大70点
- `correct: true` は **8点以上**
- `correction` は常に `null`（模範解答は別途表示）
- 採点時に **構造分解（parts）と nuance** を参照してフィードバック
- score 別の feedback ルール：
  - 10点：「正解！」のみ
  - 8〜9点：軽微な誤り + 模範の語順がなぜ望ましいか
  - 7点以下：3〜4文（誤り指摘 → 解答の和訳 → 意図とのずれ → 模範の理由）
  - 未入力：0点、具体的指摘のみ

### 1.6 採点後のUI表示（全STEP共通）

各問について以下を表示：

1. あなたの解答
2. 模範解答
3. AI評価（スコア / 正誤 / feedback）
4. **骨格フレーム**（`X · V · Y · Z` の並び）
5. **模範解答（語順のまま色分け）**（ネストは枠で囲む）
6. **各チャンクの役割**（ラベル + `n` メモ + 内部構造）
7. **模範解答のポイント**（`nuance`）

色定義は `roles.js` が唯一の正：

| 役割 | 色 |
|------|-----|
| X 名詞役 | 青 |
| V 動詞 | 緑 |
| Y 形容詞役 | 橙 |
| Z 副詞役 | 紫 |

### 1.7 技術的制約

| 項目 | 値 |
|------|-----|
| モデル | `claude-haiku-4-5-20251001` |
| 1セットの問題数 | 7問 |
| 生成 max_tokens | 8192 |
| 採点 max_tokens | 4096 |
| JSON返却 | 配列のみ、prefill `[` で安定化 |
| 状態永続化 | なし（セッション内のみ） |

## 2. Step 3 — 動詞の変化

### 2.1 ステップ概要

| 項目 | 内容 |
|------|------|
| サブタイトル | 動詞の変化 |
| 説明文 | 時制・相（進行/完了）・態（受動）・助動詞 |
| focus | 動詞の活用（現在/過去/完了/進行/受動）と助動詞（can/must/will） |
| 教科書対応 | 第6章（動詞の5変化）・第9章（動詞が背負う4情報） |
| 課題への効き目 | 課題①：骨格の正確化 |

### 2.2 この Step で訓練すること

日本語の以下の情報を、英語の動詞形に正確にマッピングする：

| 日本語の手がかり | 英語で表現する情報 |
|-----------------|-------------------|
| いつ（今・昨日・すでに） | 時制 |
| どんな様子で（〜している・〜していた） | 相（進行・完了） |
| する / される | 態（能動・受動） |
| できる・しなければ・〜だろう | 助動詞・法 |

### 2.3 扱う文法パターン（細目）

#### 2.3.1 時制

- 現在形（習慣・一般真理・三単現）
- 過去形
- 未来（will + 原形）※シード例では限定的

#### 2.3.2 相（Aspect）

- 進行形：`be + -ing`（現在進行 / 過去進行）
- 完了形：`have/has + 過去分詞`
- 完了進行：`have been + -ing`（Step 3 では優先度低めだが許容）

#### 2.3.3 態（Voice）

- 能動態
- 受動態：`be + 過去分詞`
- 完了 + 受動の複合：`has been written` など

#### 2.3.4 助動詞

- can / could（能力・可能）
- must / should（義務・推奨）※シードでは can が中心
- will / would
- 否定形（cannot, don't 等）

#### 2.3.5 副詞との共存

- 時の副詞（now, yesterday）→ **Z**
- 時の前置詞句（at 9, every morning）→ **Z**
- 副詞の中置（already in "has already been written"）→ V に含めるか Z として分離（シードでは V に含む例あり）

### 2.4 parts 分解の期待

- **V チャンク** に時制・相・態・助動詞をまとめて含める
  - 例：`is reading` / `has already been written` / `can speak` / `is opened`
- 主語・目的語は **X**
- 時・場所・様態の修飾は **Z**（動詞の周辺）

### 2.5 シード問題が示す出題バリエーション

| # | 日本語の要点 | 文法ポイント |
|---|-------------|-------------|
| 1 | 今、本を読んでいる | 現在進行 + 時の副詞（now） |
| 2 | 昨日、問題を解決した | 過去形 + 時の副詞（後置） |
| 3 | 報告書はすでに書かれている | 現在完了受動 + already 中置 |
| 4 | 3か国語を話せる | 助動詞 can + 原形 |
| 5 | 毎朝9時に開けられる | 現在形受動 + 時の前置詞句 |

### 2.6 問題生成時の Step 固有制約

- 1セット7問で **時制・相・態・助動詞のパターンをバラけさせる**
- 似た構文（例：「毎日〜することで」系）の連続を避ける
- 動詞の変化が学習の主役であること（X/Y/Z の複雑なネストは最小限に）

### 2.7 採点時の重点

- 時制の誤り（過去 vs 現在 vs 完了）
- 進行・完了の選択ミス
- 受動態の漏れ・誤用
- 助動詞 + 原形の形
- 三単現の s

## 3. Step 4 — 準動詞・前置詞句

### 3.1 ステップ概要

| 項目 | 内容 |
|------|------|
| サブタイトル | 準動詞・前置詞句 |
| 説明文 | 不定詞・動名詞・分詞・前置詞句が X / Y / Z の3役割に登場 |
| focus | 不定詞（to do）・動名詞（-ing）・分詞（doing/done）・前置詞句の3役割（X/Y/Z） |
| 教科書対応 | 第11・12・13・14・17章 |
| 課題への効き目 | 課題②：句が3役割に登場する最頻出パーツ |

### 3.2 この Step で訓練すること

**同じ形（不定詞・-ing・前置詞句）が X / Y / Z のどれになるか** を、文中の位置と働きで判定し、翻訳時に正しい語順で配置する。

> 形ではなく、位置と働きで役割が決まる（第14章の核心）

### 3.3 扱う文法パターン（細目）

#### 3.3.1 動名詞（-ing）→ 主に **X**

- 主語：`Studying English is fun.`
- 目的語：`like taking walks`
- 前置詞の目的語：`interested in learning`

#### 3.3.2 不定詞（to do）→ **X / Y / Z すべて**

| 役割 | 例 | 日本語の典型 |
|------|-----|-------------|
| X（名詞役） | To read is fun. | 〜することは |
| Y（後置修飾） | books to read | 読むべき本 / 読んでいない本 |
| Z（目的・副詞役） | went to meet him | 〜するために |

#### 3.3.3 分詞

| 形 | 役割 | 例 |
|----|------|-----|
| 現在分詞句 | Y（後置修飾） | a girl playing the piano |
| 過去分詞 | Y（後置修飾） | a book written by her |
| 過去分詞（形容詞用法） | Y/Word | looked tired |
| 分詞構文 | Z（副詞役・前置） | Walking home, he listened... |

#### 3.3.4 前置詞句 → **Y または Z**

| 位置 | 役割 | 例 |
|------|------|-----|
| 名詞の直後 | Y | the book on the desk |
| 動詞の周辺 | Z | went to the library / in the park |

#### 3.3.5 その他の Y/Z パターン

- 形容詞句（very difficult）→ Y
- 副詞句（very slowly）→ Z
- 同格（Tokyo, the capital,）→ Y
- 句動詞（gets up）→ V に含める
- 知覚動詞 + 補語（looked tired）→ V + Y

#### 3.3.6 修飾の方向ルール

| 修飾するもの | 配置 |
|------------|------|
| 単語（形容詞・分詞単独） | **前置** |
| 句（前置詞句・分詞句・不定詞句） | **後置** |
| 節 | **後置**（Step 5 で本格化） |

### 3.4 parts 分解の期待

- 準動詞・前置詞句ごとに **役割（X/Y/Z）を正しくラベル付け**
- 分詞構文は **Z** として文頭に独立チャンク
- 不定詞の Y（後置修飾）と Z（目的）は `n` で区別を明示
- 句の内部に名詞があれば `inner` でネスト（例：`in the park` 内の `the park`）

### 3.5 シード問題が示す出題バリエーション

| パターン | 例文の要点 |
|---------|-----------|
| 動名詞主語 | 英語を勉強することは楽しい |
| 分詞後置修飾（Y） | ピアノを弾いている少女 |
| 不定詞（Z・目的） | 会うために図書館に行った |
| 前置詞句（Y） | 机の上の本 |
| 不定詞（Y・後置修飾） | まだ読んでいない本 / 読むべき本 |
| 分詞構文（Z・前置） | 帰りながら音楽を聴いた |
| 過去分詞形容詞（Y） | 疲れているように見えた |
| 形容詞句（Y） | とても難しい |
| 同格（Y） | 首都の東京 |
| 副詞句（Z） | とてもゆっくりと |
| 句動詞 + 時の修飾（Z） | 毎朝6時に起きる |

### 3.6 nuance で説明すべき典型論点

- `to read`（積読）vs `should read`（義務）のニュアンス差
- 分詞構文を文頭に置く理由（同時性の自然な表現）
- `By + 動名詞` による手段表現（プロンプト例）

### 3.7 問題生成時の Step 固有制約

- **不定詞・動名詞・分詞・前置詞句が X/Y/Z の各役割で登場** するようバランス
- 同形複数役割の混同を避けるため、1問1パターンを基本に
- ネストは **1段程度**（Step 6 ほど深くしない）

### 3.8 採点時の重点

- 不定詞の役割の取り違え（目的 vs 後置修飾 vs 主語）
- 動名詞 vs 不定詞の選択
- 分詞構文の主語一致
- 前置詞句の位置（Y=名詞後 / Z=動詞周辺）
- 後置修飾の語順

## 4. Step 5 — 関係詞

### 4.1 ステップ概要

| 項目 | 内容 |
|------|------|
| サブタイトル | 関係詞 |
| 説明文 | 関係代名詞（who/which/that）・関係副詞（where）・what節：Y または X |
| focus | 関係代名詞（who/which/that）・関係副詞（where/when）・whatの名詞節 |
| 教科書対応 | 第11・13・19章 |
| 課題への効き目 | 課題③：後置修飾＝日本語との最大の差 |

### 4.2 この Step で訓練すること

日本語の **連体修飾（名詞の前）** を、英語の **後置修飾（関係詞節・名詞節）** に正しく変換する。

### 4.3 扱う文法パターン（細目）

#### 4.3.1 関係代名詞節 → **Y**（後置修飾）

| パターン | 例 |
|---------|-----|
| 主格 who/which/that | a man who speaks English |
| 目的格 that/which（省略可） | the book (that) I bought |
| 目的格 + 前置詞 | the woman I introduced to you |
| 所有格 whose | （シードでは未登場だが許容） |

#### 4.3.2 関係副詞節 → **Y**

- where（場所）
- when（時）※許容

#### 4.3.3 同格節 that → **Y**（gap なし）

- `the fact that he lied`（事実の内容を説明）

#### 4.3.4 what 節 → **X**（名詞節・目的語）

- `what he said`（先行詞を含む）
- `what to do next`（疑問詞 + 不定詞）

#### 4.3.5 関係詞節 vs 同格節の区別

| 種類 | 構造 | 判定 |
|------|------|------|
| 関係詞節 | gap あり | that が節内の空所を埋める |
| 同格節 | gap なし | 節が名詞の内容をそのまま説明 |

#### 4.3.6 inner（ネスト）の必須ケース

関係詞節の内部に骨格がある場合、`inner` で再帰分解：

```
who like taking walks in the park
  ├─ who (X)
  ├─ like (V)
  └─ taking walks in the park (X)
       ├─ taking walks (X)
       └─ in the park (Z)
```

### 4.4 日本語（jp）の書き方ルール（Step 固有）

- **連体修飾を使う**：○「昨日買った本は」 ×「この本は、昨日買った著者による」
- 英語語順のカンマ区切り直訳を避ける

### 4.5 シード問題が示す出題バリエーション

| パターン | 例文の要点 |
|---------|-----------|
| 関係代名詞（目的格） | 昨日買った本は面白い |
| 関係代名詞（主格） | 英語を流暢に話す男性 |
| 関係副詞 where | 住んでいる街 |
| what 名詞節 | 彼が言ったことを理解できなかった |
| 関係代名詞省略 | 紹介した女性は上司 |
| that 節（Y・義務） | 読むべき本（should read） |
| ネスト付き関係詞節 | 公園を散歩するのが好きな高齢者 |
| 同格節 that | 嘘をついたという事実 |
| 疑問詞+不定詞 | 次に何をすべきかわからない |

### 4.6 nuance で説明すべき典型論点

- `to read`（積読・任意）vs `should read`（義務・推奨）の対比
- 関係代名詞省略の自然さ
- what 節が「〜すること」全体を名詞として扱う点

### 4.7 問題生成時の Step 固有制約

- 関係代名詞・関係副詞・what節・同格節を **バラけさせる**
- **inner を積極的に使う**（関係詞節内に動名詞句・前置詞句がネストする問を含める）
- jp は必ず連体修飾形式

### 4.8 採点時の重点

- 後置修飾の語順（名詞 + 関係詞節）
- 関係代名詞の選択（who/which/that）
- 関係代名詞の省略
- what vs that の取り違え
- 同格節 vs 関係詞節の混同

## 5. Step 6 — 節と接続・ネスト

### 5.1 ステップ概要

| 項目 | 内容 |
|------|------|
| サブタイトル | 節と接続・ネスト |
| 説明文 | 副詞節（Z）・名詞節（X）・等位接続・Y + Z のネスト |
| focus | 副詞節（when/because/if/although）・that名詞節・等位接続・関係詞と副詞節のネスト |
| 教科書対応 | 第10・12・15・18章 |
| 課題への効き目 | 課題③の本丸（ネストの再帰性） |

### 5.2 この Step で訓練すること

- 複数の節・句が **同時に骨格にぶら下がる** 構造を処理する
- 節の内部を **再帰的に X/Y/Z 分解** する
- 等位接続で **動詞・文を並列** する

**キーセンテンス**（教科書・アプリの象徴）：

```
People who work with me work hard when an issue arises.
```

骨格：`People work hard`  
+ Y（who節）が People を後置修飾  
+ Z（when節）が work hard を修飾  
+ when節の内部に `an issue (X) + arises (V)` がネスト

### 5.3 扱う文法パターン（細目）

#### 5.3.1 副詞節（Z）

| 種類 | 接続詞 | 例 |
|------|--------|-----|
| 時 | when / while / before / after | When she is tired, she goes home early. |
| 理由 | because / since / as | ...because he wants to pass. |
| 条件 | if / unless | （許容） |
| 譲歩 | although / though | （許容） |
| 目的 | so that / in order that | so that she could pass |
| 結果 | so...that | so tired that he fell asleep |
| 比較 | than | faster than I do |
| 場所 | where | （Step 5 と重複可） |

副詞節は **前置（文頭）または後置（文末）** の両方を扱う。

#### 5.3.2 名詞節（X）

| 導入 | 例 |
|------|-----|
| that節 | I know that he is honest. |
| whether節 | whether it is true |
| wh節 / 疑問詞+不定詞 | what to do next |

#### 5.3.3 等位接続

- and / but / or / so 等で **動詞を並列**
  - 例：`studied ... and passed the exam`

#### 5.3.4 分詞構文（Z）の再登場

- 同時進行：`listening to music`（studied listening to music）

#### 5.3.5 Y + Z の複合ネスト

- 関係詞節（Y）+ 副詞節（Z）が同一文に共存
- 副詞節の内部に X+V がネスト（`when an issue arises`）

#### 5.3.6 副詞として機能する語

- home, early 等 → 前置詞不要で **Z**

### 5.4 parts / inner の期待（Step 6 固有）

- **最大2段のネスト** を積極的に使う
- 副詞節には必ず `inner`（接続詞 + 内部骨格）
- 関係詞節 + 副詞節が共存する問をセットに含める
- 等位接続の動詞は **別 V チャンク** として表現（`and passed`）

### 5.5 日本語（jp）の書き方ルール（Step 固有）

- 複数節の論理関係を明示（因果・並列・条件）
  - ○「〜ので」「〜とき」「〜しながら」
  - × 論理が曖昧な「〜で、〜」
- 後節の述語は単独で意味が通るように

### 5.6 シード問題が示す出題バリエーション

| パターン | 例文の要点 |
|---------|-----------|
| 副詞節（時・前置） | 疲れているとき、早く帰る |
| 副詞節（理由・後置） | 合格したいので、勉強する |
| that名詞節（X） | 彼が正直だと知っている |
| **Y+Z ネスト** | 一緒に働く人々は、問題が起きたとき努力する |
| 分詞構文 + 等位接続 | 音楽を聴きながら勉強し、合格した |
| whether名詞節 | それが本当かどうかわからない |
| so that（目的） | 合格できるように勉強した |
| so...that（結果） | とても疲れていたので、すぐ眠った |
| 比較（than） | 私より速く走る |

### 5.7 問題生成時の Step 固有制約

- **ネスト付きの問を必ず含める**（Y+Z 複合が理想）
- 副詞節の種類（時・理由・目的・結果・比較）をバラけさせる
- 名詞節と副詞節と等位接続を混在させる
- Step 5 の関係詞パターンと組み合わせて難度を上げる

### 5.8 採点時の重点

- 副詞節の接続詞選択
- 名詞節（that/whether/wh）の役割
- 節の前置 vs 後置
- 等位接続での動詞形の一致
- ネスト構造を壊す語順の誤り
- so that / so...that の形
